# Demo Handoff

## Purpose

This repository is being prepared for external verification as a demo-safe MVP.

## Supported local runtime

- Frontend: `http://127.0.0.1:8082/app/index.html`
- Backend: `http://127.0.0.1:8787`

## Demo data profile

- random demonstration contracts only,
- random demonstration employees only,
- random demonstration invoices only,
- controlled demo months only,
- no active business data.

## Handoff checklist

Before sending the project to an external reviewer, confirm:
- supported start scripts work,
- demo login is documented,
- data is demo-only,
- contract, invoice, and time modules are consistent,
- known limitations are written down,
- UI QA has passed.

## Known current limitations

- time tracking is still completing the backend-first transition,
- some compatibility helpers still exist in older frontend layers,
- full UI polish is still planned as a dedicated stage.

## What should be included in the final handoff package

- repository link,
- supported startup steps,
- demo credentials,
- MVP scope,
- known limitations,
- latest QA result.
